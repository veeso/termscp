# 批量操作文件

termscp 允许你一次对多个文件执行操作。使用以下控制方式来构建选择。

- `<M>`：将高亮的文件标记为已选择。
- `<CTRL+A>`：选择当前目录中的所有文件。
- `<ALT+A>`：取消选择所有文件。

一旦某个文件被标记为已选择，它会以高亮背景显示。

当存在选择时，只有被选中的文件会被纳入操作处理；当前高亮的条目会被忽略。你也可以在查找结果面板中对多个文件进行操作。

## 对选择执行的操作

所有操作在处理多个文件时都可用，但其中一些行为略有不同。当存在选择时，你输入的名称指的是目标目录，而不是单个目标文件。

- **复制**：系统会提示你输入目标位置。当选中多个文件时，该名称是所有文件被复制到的目标目录。
- **重命名**：与复制相同，但文件会被移动到目标目录。
- **另存为**：与复制相同，但文件会被写入目标目录。

## 传输队列

如果你在某个目录（例如 `/home`）中选择了一个文件，然后切换目录，该文件仍会保持被选中状态，并显示在底部面板的**传输队列**中。

当某个文件被选中时，当时活动的远程目录会与其条目相关联。如果该文件随后被传输，它会被传输到与之关联的远程目录。

### 示例

假设你在远程面板位于 `/tmp` 时选择了本地文件 `/home/a.txt`，然后移动到 `/var`，在远程面板位于 `/home` 时选择了 `/var/b.txt`，最后执行传输。结果是：

- `/home/a.txt` 被传输到 `/tmp/a.txt`
- `/var/b.txt` 被传输到 `/home/b.txt`
